## Project overview

Qada is a [TanStack Start](https://tanstack.com/start) app: React 19 + TanStack Router (file-based routing) + Vite 8, styled with Tailwind CSS v4 and shadcn/ui (Radix primitives). It helps users track and make up ("qada") missed prayers and fasts — a home screen with daily counters, and a debt screen for entering how much is owed and viewing progress toward paying it off. State persists to `localStorage`.

Development follows the [Spec Kit](https://github.com/github/spec-kit) workflow: features are specified, planned, and broken into tasks under `specs/<NNN-feature-name>/` before implementation (see the `speckit-*` skills).

## Commands

```bash
npm run dev              # start dev server on port 3000
npm run build             # production build (vite build)
npm run preview           # preview the production build
npm run generate-routes   # regenerate src/routeTree.gen.ts from src/routes/*
npm run test              # run tests with vitest
npm run lint               # eslint
npm run format              # prettier --write . && eslint --fix
npm run check                # prettier --check . (no writes)
```

To run a single test file with vitest: `npx vitest run path/to/file.test.ts`.

Vitest is configured inline in `vite.config.ts` (`test: { environment: 'jsdom', globals: true }`) — there is no separate vitest config file. Tests live alongside source as `*.test.ts`/`*.test.tsx`.

## Architecture

- **File-based routing**: routes live in `src/routes/`. `src/routes/__root.tsx` defines the root document shell (`<html>`, `<head>`, `<Scripts>`), mounts the global `StorageStatusBanner`, sets meta/head tags, and mounts TanStack Devtools (router devtools panel, bottom-right). Each route file exports a `Route` via `createFileRoute('/path')({ component })` and just wires a path to a feature screen component (e.g. `/` → `HomeScreen`, `/debt` → `DebtScreen`).
- **`src/routeTree.gen.ts`** is auto-generated by `tsr generate` (aliased as `npm run generate-routes`) from the contents of `src/routes/`. Never hand-edit it — it's regenerated and is excluded from lint/format (see `.prettierignore`/eslint ignores). Regenerate it after adding/removing/renaming route files.
- **`src/router.tsx`** creates the TanStack Router instance (`getRouter`) consumed by both the client entry and SSR, with `defaultPreload: 'intent'`.
- **Features (`src/features/`)**: each feature is self-contained with its own `types.ts`, `store.ts` (state + persistence side effects), `calc.ts` (pure calculation logic, unit-tested), and `components/`.
  - `features/counters` — the home screen: per-prayer (`fajr`, `dhuhr`, `asr`, `maghrib`, `isha`) and fasting make-up counters (`HomeScreen.tsx`, `SalahSection.tsx`, `FastSection.tsx`, `PrayerRow.tsx`, `CounterButton.tsx`, `ProgressHeader.tsx`).
  - `features/debt` — entering total prayer/fast debt and viewing progress against it (`DebtScreen.tsx`, `DebtEntryForm.tsx`, `ProgressView.tsx`, `CategoryProgress.tsx`, `DateRangeEstimator.tsx`).
  - `features/storage` — `StorageStatusBanner.tsx`, a global banner reflecting `src/lib/storage` availability.
- **Persistence (`src/lib/storage/`)**: `persistence.ts` reads/writes a single `ProgressRecord` (see `progressRecord.ts`) to `localStorage` under `STORAGE_KEY`, never throwing — failures degrade to `DEFAULT_RECORD` on read and `{ ok: false }` on write. `migrate.ts` upgrades stored records to `SCHEMA_VERSION`. `status.ts` is a module-level store (`useSyncExternalStore`) tracking storage availability, flipped by `markUnavailable()` and surfaced by `StorageStatusBanner`. `updateRecord()` is the shared atomic read-modify-write used by both the counters and debt stores; same-device cross-tab sync goes through `subscribeExternal` (window `storage` event).
- **i18n (`src/i18n/`)**: `en.ts` holds the flat message dictionary; `index.ts` exposes `t(key, vars)` (interpolates `{var}` placeholders), `plural(unit, n)`, and `useDirection()` (currently always `'ltr'`, single `en` locale — treat as the seam for future locales/RTL rather than assuming it's permanent).
- **UI primitives (`src/components/ui/`)**: shadcn/ui-style components (`button.tsx`, `input.tsx`, `label.tsx`, `progress.tsx`) built on Radix (`radix-ui`, `@radix-ui/react-dialog`, `@radix-ui/react-slot`) and `class-variance-authority`/`clsx`/`tailwind-merge`. `src/lib/cn.ts` is the `clsx` + `tailwind-merge` class-merging helper used throughout.
- **Styling**: Tailwind v4 via the `@tailwindcss/vite` plugin (no `tailwind.config.js` — v4 uses CSS-based config). Global styles/import in `src/styles.css`, injected in the root route via `?url` import.
- **Path aliases**: both `#/*` and `@/*` resolve to `./src/*` (see `imports` in `package.json` and `paths` in `tsconfig.json`). Prefer these over relative `../../` imports across route/module boundaries.

## Code style

- Prettier: no semicolons, single quotes, trailing commas everywhere (`prettier.config.js`).
- ESLint config extends `@tanstack/eslint-config` (`eslint.config.js`), with these rules explicitly disabled: `import/no-cycle`, `import/order`, `sort-imports`, `@typescript-eslint/array-type`, `@typescript-eslint/require-await`, `pnpm/json-enforce-catalog`.
- TypeScript: `strict` mode plus `noUnusedLocals`, `noUnusedParameters`, `noFallthroughCasesInSwitch`, `verbatimModuleSyntax`. Module resolution is `bundler`; `.ts`/`.tsx` extensions are allowed in imports.
